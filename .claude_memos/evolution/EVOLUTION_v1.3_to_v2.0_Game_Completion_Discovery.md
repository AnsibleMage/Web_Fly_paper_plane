## 🛠️ 미래 AI를 위한 학습 자료

            ↑

**올바른 패턴:**
```
문서 읽기 → 추론 → **실제 코드 검증** → 계획
                        ↑
                    필수 단계
```

**적용 방법:**
1. PRD/설계 문서 읽기 ✅
2. **핵심 파일 읽기** (App.tsx, main entry point) ✅
3. **실행 테스트** (npm run dev) ✅
4. 추론 검증 ✅
5. 계획 수립 ✅

### Lesson 4: 시간 압박 vs 정확성 트레이드오프

**15:00 세션 (잘못된 선택):**
```
30분 제한 → 문서만 분석 → 빠른 완성 → 70% 오차
             ↑                          ↓
          검증 생략                 5시간 오해 지속
```

**20:20 세션 (올바른 선택):**
```
30분 할당 → 파일 읽기 + 실행 → 정확한 파악 → 올바른 계획
             ↑                              ↓
          검증 포함                    3.5일 단축
```

**교훈:**
- "빠른 분석" < "정확한 분석"
- **검증 생략 = 더 큰 시간 낭비**
- 10분 검증 투자 → 3일 절약

**적용 방법:**
1. 시간 압박 상황
2. "검증 생략" 유혹 발생
3. **10분 투자 = 나중에 10배 절약** 상기
4. 핵심 파일 읽기 우선 순위
5. 실행 테스트 포함

### Lesson 5: Canvas 기반 게임의 E2E 테스트 한계

**문제:**
```
Playwright E2E → Canvas 내부 테스트 불가
                ↓
             WebGL 렌더링 검증 실패
                ↓
            false negative 발생
```

**해결:**
```
E2E 시나리오 재설계
  ↓
Canvas 외부 UI 테스트 (Result Screen, Leaderboard)
  ↓
window.gameFailed, window.gameFinished 플래그 검증
  ↓
간접 검증 (게임 → 결과 화면 전환)
```

**적용 방법:**
1. Canvas/WebGL 게임 E2E 작성 시
2. **직접 렌더링 테스트 불가** 인지
3. 전역 플래그 노출 (`window.*`)
4. 화면 전환 검증 (Menu → Game → Result)
5. 간접 검증 패턴 사용
