### 1. Zustand Store Mocking 패턴 확립 (중요!)

**문제:**
- Leaderboard 테스트에서 동적 상태 변경 불가
- 초기 mock 구현이 immutable 객체 반환
- Loading 상태 변경 테스트 실패

**원인:**
```typescript
// ❌ 문제 코드: 상태 변경 불가
const mockState = { entries: [], isLoading: false };
vi.mock('../../../stores/socialStore', () => ({
  useSocialStore: () => mockState,
}));
```

**해결:**
```typescript
// ✅ 해결: let 변수 + beforeEach 리셋
let mockState = { entries: [], isLoading: false, /* ... */ };

beforeEach(() => {
  mockState = { /* 초기 상태 */ };
});

vi.mock('../../../stores/socialStore', () => ({
  useSocialStore: (selector?: any) => {
    return selector ? selector(mockState) : mockState;
  },
}));

// 테스트 중 상태 변경 가능
mockState.isLoading = true; // ✅ 동작함
```

**교훈:**
- **Zustand store mocking 시 `let` 변수 사용 필수**
- Selector 패턴 정확히 모방
- `beforeEach`로 테스트 간 격리

**앞으로:**
- 모든 Zustand store mock에 이 패턴 적용
- 상태 변경이 필요한 테스트는 `let mockState` 사용
- 패턴을 테스트 템플릿화

### 2. Integration Test Selector 전략

**문제:**
- "Settings" 텍스트가 버튼과 heading에 중복
- `getByText('Settings')` 실패: "Found multiple elements"

**원인:**
```tsx
// 버튼에 "Settings"
<button>Settings</button>

// 패널 heading에도 "Settings"
<h2>Settings</h2>
```

**해결:**
```typescript
// ❌ 문제: 중복 텍스트
expect(screen.getByText('Settings')).toBeInTheDocument();

// ✅ 해결: 더 구체적인 selector
expect(screen.getByRole('heading', { name: /settings/i })).toBeInTheDocument();
```

**교훈:**
- **Integration 테스트는 더 구체적인 selector 사용**
- `getByRole` > `getByText`
- 의미론적 HTML 활용

**앞으로:**
- 통합 테스트에서 role-based selector 우선
- 중복 가능성 있는 텍스트는 role + name 조합

```typescript
// Helper 함수 추출

  const button = screen.getByRole('button', { name });
  fireEvent.click(button);
  return button;
};

const measureResponseTime = (action: () => void): number => {
  const startTime = performance.now();
  action();
  return performance.now() - startTime;
};

const openSettingsPanel = () => clickButton(/settings/i);
const openLeaderboardPanel = () => clickButton(/leaderboard/i);
```

### 1. Zustand Store Mocking 패턴 확립 ⭐⭐⭐ (가장 중요!)

**발견:**
- 초기 mock 구현이 immutable 객체 반환
- 동적 상태 변경 테스트 불가
- Loading 상태 전환 테스트 실패

**교훈:**
- **`let mockState` 변수 + `beforeEach` 리셋 필수**
- Selector 패턴 정확히 모방
- 상태 변경 가능한 참조 유지

**패턴:**
```typescript
let mockState = { /* 초기 상태 */ };

beforeEach(() => {
  mockState = { /* 리셋 */ };
});

vi.mock('store', () => ({
  useStore: (selector?: any) => {
    return selector ? selector(mockState) : mockState;
  },
}));
```

**앞으로:**
- 모든 Zustand store mock에 적용
- 테스트 템플릿화
