## 주요 결정사항 및 제약조건

1. **4-Layer 프랙탈 TDD 아키텍처 (v1.1):**
   - Layer 0: Product (E2E Test, 2-4주)
   - Layer 1: Block (Module Test, 3-7일)
   - Layer 2: Feature (Integration Test, 1-2일)
   - Layer 3: Task (Unit Test, 1-2시간)
   - **계층 구조:** 1 Product = 3 Blocks = 9 Features = 45 Tasks

2. **피라미드 워크플로우 (v1.1):**
   - Task (개발 중 ⬇️): Unit Test 동시 작성
   - Feature (통합 후 ⬆️): Integration Test 작성
   - Block (모듈 후 ⬆️): Module Test 작성
   - Product (E2E 후 ⬆️): E2E Test 작성 (모든 Block 완료 후)

3. **PRD 싱크 포인트 3단계:**
   - Feature Integration Test 완료 후 → PRD 싱크
   - Block Module Test 완료 후 → PRD 싱크
   - Product E2E Test 완료 후 → PRD 최종 검증

4. **환경 분리 전략 (신규):** ⭐⭐⭐ NEW!
   - **Windows Obsidian**: 문서 작업만 (PRD, 연구, 회의록)
   - **macOS 개발 환경**: Block 설계 + 코드 개발
   - **이유**: Block 설계는 폴더 구조와 밀접, 코드 작성과 유사
   - **효과**: 환경 일관성 유지, AI 파일 탐색 최적화

5. **AI-Optimized 폴더 구조:**
   - Block/Feature/Task 계층 = 폴더 계층
   - 네이밍 패턴: `block[N]-[name]/features/f[N]-[name]/tasks/t[N]-[name].ts`
   - AI가 PRD 보고 < 0.1초에 파일 탐색 가능

6. **Development Kickoff 자동화 (5단계):**
   - 1단계: 프로젝트 초기화 (Git, npm)
   - 2단계: AI-optimized 폴더 구조 생성
   - 3단계: 라이브러리 & 에셋 설치
   - 4단계: 기본 설정 파일 생성
   - 5단계: Block 1 진입점 준비 (Red test)

7. **AI 메모리 시스템:**
   - Session Memo: 모든 세션 종료 시 자동 기록 (1-2분)

   - 트리거 키워드: "메모해줘" / "기억해줘"

8. **2인 개발팀 워크플로우:**
