## 💡 핵심 학습 (Key Learnings)

### 1. Block Store 확장 패턴
**문제:**
- Block 3의 socialStore에 currentUser 필드가 없음
- Integration test 실패: `setCurrentUser is not a function`

**해결:**
```typescript
// Block 3 store를 기반으로 확장
const baseStore = createLeaderboardStore();

export const useSocialStore = create<ExtendedSocialStore>((set) => ({
  ...baseStore.getState(),  // Block 3 기본 상태
  currentUser: null,        // Block 4 확장 상태
  setCurrentUser: (user) => set({ currentUser: user }),
  // Block 3 액션 재정의
}));
```

**교훈:**
- Block 간 store 재사용 시 확장 패턴 적용
- 기존 기능 유지하면서 새 필드 추가
- gameStore, timerStore에서도 동일 패턴 사용

### 2. Integration Test cleanup() 패턴
**문제:**
- 시나리오 9에서 중복 render() 호출
- "Found multiple elements with the text: New Record!" 에러

**해결:**
```typescript
// 이전 렌더링 제거
cleanup();

// 새로운 상태로 재렌더링
setFinalTime(60000);
setPersonalBest(70000);
render(<ResultScreen />);
```

**교훈:**
- 복합 시나리오에서 cleanup() 명시적 호출
- rerender() 대신 cleanup() + render() 사용
- DOM 상태 완전히 초기화
